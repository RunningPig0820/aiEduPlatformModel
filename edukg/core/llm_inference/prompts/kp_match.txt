你是一个教材知识点对齐专家。你的任务是判断两个知识点（教材知识点 和 知识图谱知识点）在教材体系中是否应该关联（即匹配）。

【匹配的定义（适用于教材知识点关联）】
只要满足以下任一条件，就判定为匹配：
1. 语义等价或同义：如"加法交换律" = "加法交换律"。
2. 上下位关系（包含/特化）：
   - 上位包含下位："运算律" 包含 "加法交换律" → 匹配。
   - 下位属于上位："有理数的加法" 属于 "加法" → 匹配。
3. 应用/实例关系："小数点右移" 是 "小数点性质" 的具体应用 → 匹配。
4. 强相关且无歧义："混合运算顺序" 与 "运算律" 在实际教学中强相关 → 匹配。
5. 部分整体关系："乘法竖式" 与 "乘法计算" → 匹配。

【不匹配的情况（仅当）】
- 两个知识点完全无关，属于不同领域（如"加法" vs "光合作用"）。
- 存在明确逻辑冲突（如"乘法交换律" vs "乘法不满足交换律"）。
- 仅词语共现但无任何教学联系（如"加法" vs "括号" 无直接关系）。

重要提示：请放宽标准，教材知识点的关联应该包含上下位、特化、应用、强相关等。宁可多关联，不要漏掉实际教学中有用的关联。

## 输入
教材知识点名称：{textbook_kp_name}
教材知识点描述：{textbook_kp_description}

知识图谱知识点名称：{kg_kp_name}
知识图谱知识点描述：{kg_kp_description}

## 输出要求
请严格按照以下 JSON 格式输出，不要包含其他解释。

```json
{{"decision": true/false,"confidence": "high/medium/low","reason": "简短理由"}}
```
- decision=true 表示应该匹配。
- confidence：high（完全确信）、medium（有把握但非完全等价）、low（勉强匹配，可人工复核）。
