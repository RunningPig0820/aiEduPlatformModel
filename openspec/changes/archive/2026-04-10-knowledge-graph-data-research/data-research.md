| 学科 | 知识点数 | relateTo | subCategory | 关系数据 |

| 化学 | 5,718 | 0 | 0 | ❌ 无 |
| 生物 | 15,209 | 0 | 0 | ❌ 无 |
| 历史 | 4,850 | 0 | 0 | ❌ 无 |
| 语文 | 8,041 | 0 | 0 | ❌ 无 |
| 地理 | 4,682 | 0 | 0 | ❌ 无 |
| 政治 | 5,309 | 0 | 0 | ❌ 无 |
| 英语 | 5,107 | 0 | 0 | ❌ 无 |

**结论**: 只有数学有关系数据，其他学科全部缺失！

### 1.2 不同学科知识点特点差异

| 学科 | 知识点特点 | 前置依赖特点 | 数据可用性 |
|------|-----------|-------------|-----------|
| **数学** | 概念、定义、定理、公式 | **强逻辑链**，学习顺序明确 | ✅ 有类型分类 + 关系数据 |
| **物理** | 概念、定律、现象 | 较强逻辑链（力学→电磁学） | ⚠️ 有知识点，无关系 |
| **化学** | 元素、反应、原理 | 有一定逻辑链 | ⚠️ 有知识点，无关系 |
| **生物** | 细胞、遗传、生态 | 有一定逻辑链 | ⚠️ 有知识点，无关系 |
| **历史** | 人物、事件、制度 | **时间线/主题关联**，非学习依赖 | ⚠️ 有知识点，无关系 |
| **语文** | 作家、作品、手法 | **主题关联**，非学习依赖 | ⚠️ 数据质量存疑 |
| **英语** | 词汇、语法 | 有一定逻辑链 | ⚠️ 有知识点，无关系 |

## 二、回答你的问题

### Q1: 设计能否满足 AI 答疑、学生知识点梳理需求？

**答: 部分满足**

| 需求 | 数学 | 其他学科 |
|------|------|---------|
| 识别题目知识点 | ✅ 可以 | ✅ 可以 |
| 查询知识点学科 | ✅ 可以 | ✅ 可以 |
| 查询知识点年级 | ⚠️ 需推断 | ⚠️ 需推断 |
| 查询前置知识 | ⚠️ 有基础数据 | ❌ 需要构建 |
| 学习路径推荐 | ⚠️ 可以 | ❌ 缺数据 |
| 知识缺陷诊断 | ⚠️ 可以 | ❌ 缺数据 |

### Q2: 不同学科上下文关联是否不同？

**答: 是的，差异很大**

```
数学/物理/化学: 强逻辑链，学习顺序明确
  例: 加法 → 乘法 → 幂运算 → 指数函数
  适合: "学习A需要先掌握B" 的前置关系

历史: 时间线/主题关联，不是学习依赖
  例: 明朝 → 清朝（时间顺序，但学清朝不需要先学明朝）
  适合: 时间线、主题分类

语文: 主题/作品关联
  例: 鸿门宴 → 司马迁（作者关联）
  适合: 主题分类、作品关联
```

### Q3: main.ttl 有多个学科，为什么只处理数学？

**答: 这是设计疏忽**

main.ttl 包含 8 个学科的数据：
- biology: 1,367
- geo: 1,243
- chemistry: 1,138
- history: 1,035
- physics: 1,029
- math: 784
- politics: 702
- chinese: 677

应该全部处理，但策略不同：
- **数学/物理/化学**: 重点构建前置依赖
- **历史/语文**: 重点构建主题/时间分类

### Q4: 如何验证设计正确性？

**验证方案**:

1. **数据验证**
   - 抽查前置关系的合理性
   - 由教师/教研审核

2. **功能验证**
   - 构建测试用例
   - 验证学习路径推荐是否合理

3. **业务验证**
   - 小范围试点
   - 收集用户反馈

## 三、修订后的设计

### 3.1 分学科策略

| 学科类型 | 学科 | 策略 | 重点 |
|---------|------|------|------|
| **强逻辑链学科** | 数学、物理、化学、生物 | 构建前置依赖 | 学习路径、知识诊断 |
| **主题关联学科** | 历史、语文 | 构建主题分类 | 知识关联、主题检索 |
| **语言学科** | 英语 | 构建语法/词汇层级 | 学习路径 |

### 3.2 数据整理优先级

```
Phase 1: 数学（有关系数据，验证设计）
Phase 2: 物理、化学、生物（有前置关系需求，需构建）
Phase 3: 英语（有语法层级）
Phase 4: 历史、语文、地理、政治（主题分类为主）
```

### 3.3 验证方案

```python
# 方案 1: 自动验证
def validate_prerequisite(prerequisite, subject):
    """验证前置关系的合理性"""
    # 规则检查
    - 不应该有循环依赖
    - 年级不应该倒置（高一知识 不应该是 高三知识 的前置）
    - 同类型知识优先级（定义 → 性质 → 定理）

# 方案 2: 人工抽查
- 随机抽取 100 条前置关系
- 由教研老师判断是否合理
- 计算准确率

# 方案 3: 用户反馈
- 提供反馈按钮

- 持续优化
```
